Chat Request — Data Flow
1 Browser User types a message; app.js calls apiFetch("/api/chat", body)
2 chat.py Extracts message, profile_id, history; fetches profile from db.py
3 watsonx_client Injects AGENT_INSTRUCTIONS + profile context; builds Granite prompt
4 Granite-4 Generates NutriBot reply via generate_text()
5 Browser JSON {success, data.reply} rendered in chat UI
API Endpoints
Method Path Blueprint Description
POST /api/chat chat.py Conversational nutrition agent reply (+ optional profile context)
POST /api/nutrition/analyse nutrition.py Calorie & macro breakdown for a list of food items (JSON)
POST /api/meal-plan/generate nutrition.py Generate 1–7 day personalised meal plan for a saved profile
POST /api/bmi bmi.py Compute BMI, category, healthy weight range (no AI)
GET /api/profiles profiles.py List all family profiles
GET /api/profiles/:id profiles.py Fetch a single profile
POST /api/profiles profiles.py Create a new family member profile
PUT /api/profiles/:id profiles.py Update an existing profile
DELETE /api/profiles/:id profiles.py Remove a profile
Database Schema — nutrition.db
TABLE family_profiles
id PK INTEGER AUTOINCREMENT primary key
name TEXT NOT NULL
age INTEGER NOT NULL
gender TEXT NOT NULL
weight_kg REAL NOT NULL
height_cm REAL NOT NULL
activity_level TEXT DEFAULT 'Moderately Active'
dietary_restrictions TEXT DEFAULT '' — comma-separated flags
health_goals TEXT DEFAULT '' — free text
created_at TEXT ISO-8601 UTC timestamp
